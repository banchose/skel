# Bash Scripting Standards

## Non-Negotiable Rules
- **Always quote variables**: `"${var}"`, not `$var`
- Use `set -euo pipefail` at the top of every script
- Use `[[ ]]` for conditionals, never `[ ]`
- Brace-wrap all variable references: `"${MY_VAR}"`, not `"$MY_VAR"`
- Use `readonly` for constants
- Use `local` for function variables

## Error Handling
- Trap errors: `trap 'echo "Error on line $LINENO" >&2; exit 1' ERR`
- Check command existence before use: `command -v foo >/dev/null 2>&1`
- Redirect error messages to stderr: `echo "error" >&2`

## Style
- Use lowercase for local variables, UPPERCASE for exported/environment variables
- Use `printf` over `echo` for anything non-trivial
- Use `$(command)` for substitution, never backticks
- Heredocs over long strings of `echo`

## Example
#!/usr/bin/env bash
set -euo pipefail

readonly REGION="${1:?Usage: $0 <region>}"

main() {
    local instance_id
    instance_id="$(aws ec2 describe-instances \
        --region "${REGION}" \
        --query 'Reservations[0].Instances[0].InstanceId' \
        --output text)"

    printf 'Instance: %s\n' "${instance_id}"
}

main "$@"
